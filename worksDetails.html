Iterative Design

Mockup for Reach.live

Overview

Iterative prototyping is an essential part of creating user interfaces. In this group project, my teammates and I designed an interface for a startup from scratch using only their concept. We did not use anything they‚Äôve already built such as logo, images, or content. Afterwards, we iterated on our design based on feedback from classmates and an industry critic, and tested our final prototype on real users through usertesting.com.

Timeline: 2 weeks

Tools: Figma, VS Code

Context: Brown CS1300 Iterative Design Group Project

Context

I had 3 teammates on this project. Iteration Process

RESEARCH-> SKETCH-> WIREFRAME-> HIGN FI ITERATIONS-> INTERVIEWS-> CONCLUSION

RESEARCH

Understand the product & define target users
Product & Target Users

Our target user group is lifestyle creators, especially those hoping to profit from their contents and discover contents from fellow creators. Since these creators mostly produce and livestream contents on their computers, we decided to create a web interface to enable easier access and more flexible layouts.

SKETCH & WIREFRAME

Brainstorm solutions & products
Phase 1. Sketch

Each member of our group drew a set of sketches for at least three different pages: home page, event page, and analytics page. Our design for the event pages were very diverse as demonstrated below.

Sketch

Phase 2. Wireframes

After sketching ideas on paper, we created wireframes by combining different ideas. A design feature I want to highlight is the landing pages. Before users register, they can only see the home and explore events page. To see the rest of the website, they need to signin. We believed this design would prompt more users to register for the site. Since the company is in its startup stage, having more registered users would benefit scale and profitability.

Home & Explore Events (Prelogin)

Home Page

We want to show Reach.live is a platform for users to both host and participate in events. Therefore, we demonstrated those through sections of discover events, host events, monetize events, and live interactions.

Explore Events Page

The Explore Events page shows a set of featured events under each category. We chose to present all categories on one page without using any filter or tabs because this would make the page look more straightforward and require fewer interactions, thus creating a more effortless experience for first-time users.

Events & Analytics

Events Page

There are two sections on the Events page: 1. Discover Events 2. My Events. In the Discover Events section, users see events in the default category, and they can view events of other categories when they click on tabs. We believe this design helps users discover intersting events more easily.

Analytics Page

The analytics page contains a dashboard displaying a summary of important metrics, and several charts displaying detailed breakdowns of each metric. We made this design decision because we felt it was important for creators to gain insights into events they have hosted.

Account & Host Events

Account Page

The Account page enables user to view and edit important information about user profile, password, email, and payments which needs to be viewed or modified on a regular basis.

Host Events Pop-Up

The host events button is located on the top of all main pages for easy access. Content creators can input information such as what the event is about, time of the event, and how much the event costs.

Phase Two. Feedback & Iterate.

During studio, we received many helpful suggestions. And we incorporated them in our 2nd iteration through the following features and elements.

ITERATION 2: Logout Button

Feedback: Website does not provide an easy way to logout or switch accounts.

Solution: Added a sign out link on the navigation bar right beside ‚ÄúWelcome, Emily‚Äù to improve usability.

ITERATION 2: Event Filters

Feedback: Users can‚Äôt easily filter events based on timing or date.

Solution: Designed filters in the main page to help users identify events based on the type, the date, and time of day through 3 drop down menus .

ITERATION 2: Monetization Insights

Studio Feedback: Industry critic felt a need to gain more qualitative insights about events users have hosted in addition to numbers.

Solution: In the analytics page, we added an ‚ÄúInsight‚Äù section to help creators learn and succeed on the platform.

Accesibility

We were cautious of potential accessibility issues during our design stage. Especially for the graphs on our analytics page, our team made sure that there are enough colour contrasts in the pie charts and data charts.

Analysis

We expected the users to be able to complete the sub-tasks easily without any confusion given our intentional design of clear menu options, minimum number of buttons, and intuitive layout. The results from user testing aligned with our expectations and turned out to be very positive. One common thing they all mentioned was that the website is easy to navigate, simple, clear and well-put together. When watching their interactions with our site, we recognized that they all liked its modern and personal feel and appreciated its clear layout. The responses and comments from the three users are summarized below.

Sub Tasks

In general, the users not only found all four tasks easy to complete, but were also confident that they had completed the tasks successfully. The durations of time they spent on each task were very similar and no one seemed to struggle with a particular task. Though none of the users made an error, there were few moments when some users felt slightly unsure about their actions.

CONCLUSION

Takeaway and what I learned

This project was an invaluable opportunity for me to work in a team and practice the entire iterative design process. By combining diverse ideas, feedback from classmates and industry experts, I learned various good practices of designing a clear, intuitive interface such as the importance of a coherent visual identity and clear navigation.
